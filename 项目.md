# Windows 桌面效率工具项目开题

## 一、项目名称

**DeskHub：Windows 桌面效率与轻量游戏工具**

---

## 二、项目背景

日常使用 Windows 电脑时，用户经常需要同时处理任务、邮件、桌面快捷方式和工作提醒。
这些内容分散在不同软件中，管理成本较高。

本项目计划开发一款 Windows 桌面产品，将 Todo 任务管理、桌面图标管理、邮件管理、背景美化和挂机小游戏结合在一起，形成一个轻量化的桌面效率中心。

---

## 三、项目目标

本项目目标是开发一款低内存、性能较好的 Windows 桌面应用。

主要目标包括：

* 管理每日 Todo 任务
* 整理和启动桌面快捷方式
* 查看和管理邮箱邮件
* 从互联网搜索并下载自然风景图片作为背景
* 通过挂机小游戏增强使用动力
* 支持本地数据保存
* 支持系统托盘和后台运行

---

### 5. 挂机小游戏

## 五、技术选型

本项目推荐使用：

```txt
Tauri + Svelte + Rust + SQLite
```

### 技术说明

| 技术         | 用途             |
| ---------- | -------------- |
| Tauri      | 桌面应用框架         |
| Svelte     | 前端界面开发         |
| TypeScript | 前端逻辑           |
| Rust       | 本地后端和系统调用      |
| SQLite     | 本地数据库          |
| windows-rs | Windows API 调用 |
| reqwest    | 网络请求和图片下载      |

选择原因：

* 比 Electron 更轻量
* 内存占用更低
* Rust 性能较好
* Svelte 适合开发轻量 UI
* SQLite 适合本地数据保存
* 适合长期后台运行
* 方便调用 Windows 系统能力

---

## 六、系统架构

```txt
DeskHub
├── 前端界面：Svelte + TypeScript
├── 桌面框架：Tauri
├── 本地后端：Rust
├── 数据存储：SQLite
├── 系统调用：windows-rs
├── 网络请求：reqwest
└── 第三方服务：Gmail API / Microsoft Graph API / 图片搜索 API
```

## 八、MVP 范围

第一版只实现核心功能：

* Todo 新增、删除、完成
* 本地 SQLite 保存
* 挂机金币系统
* Todo 完成奖励金币
* 搜索自然风景图片
* 下载并设置应用背景
* 系统托盘
* 关闭后重新打开数据仍然存在

邮箱和完整桌面图标管理可以放到后续版本开发。

## 九、项目难点

* Windows 快捷方式和图标读取
* 邮箱 OAuth 登录和 Token 管理
* 图片搜索 API 接入
* 图片版权和来源记录
* 本地数据安全保存
* 后台运行时的内存控制
* 游戏系统和效率工具的平衡

## 十、背景图片版权说明

背景图片下载需要注意版权问题。

第一版建议只使用：

* 免费授权图片
* 可商用图片
* 允许下载和本地使用的图片来源

应用需要保存图片来源链接，方便后续查看和管理。
