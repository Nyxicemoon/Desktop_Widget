# DeskHub 开发计划

> 本文档基于 [项目.md](项目.md) 的需求规格，将其细化为可执行的迭代计划。
> 技术栈：Tauri + Svelte + TypeScript + Rust + SQLite（详见 [CLAUDE.md](CLAUDE.md)）。
> 原则：本地优先（local-first）、低内存、可后台长期运行。

---

## 一、总体路线图

| 阶段 | 主题 | 目标产物 | 预估周期 |
| -- | -- | -- | -- |
| M0 | 工程脚手架 | 可运行的空壳应用、CI、数据库迁移框架 | 1 周 |
| M1 | Todo + 金币（**MVP 核心**） | Todo 增删改完成 + 离线持久化 + 基础金币系统 | 2 周 |
| M2 | 背景图片管理 | 在线搜图、下载、设为应用背景、来源记录 | 1.5 周 |
| M3 | 系统集成与打包 | 系统托盘、开机自启、本地通知、Windows 安装包 | 1.5 周 |
| M4 | 桌面图标管理 | 扫描快捷方式、分类收藏、一键启动 | 1.5 周 |
| M5 | 邮件管理 | Gmail OAuth、邮件列表/详情/搜索/已读状态 | 2.5 周 |
| M6 | 挂机游戏深化 | 经验等级、自动产出、离线收益、行为奖励 | 1.5 周 |

> 邮件（M5）与完整桌面图标管理（M4）按规格放入后续版本。
>
> **产品形态确认（2026-06-16）：** 最终形态为**桌面透明小组件**（gadget 风格）+ 主窗口。已完成「桌面透明 Widget（Todo/金币）」里程碑：多窗口、透明、bottommost+可交互（Win32）、拖动、位置与显隐持久化。托盘控制 widget 显隐留到 M3。详见 [spec](docs/superpowers/specs/2026-06-16-deskhub-desktop-widgets-design.md)。

## 二、阶段详解

### M0 — 工程脚手架（地基）

**目标：** 一个能 `npm run tauri dev` 启动的空应用，具备数据库迁移、前后端通信约定与基础质量门禁。

- [x] `npm create tauri-app` 初始化（实际为 SvelteKit + TypeScript 模板，Tauri v2），落到 `src-tauri/`
- [x] 配置目录结构：
  - `src/` 前端（`lib/api/`、`lib/stores/`、`routes/`）
  - `src-tauri/src/` 后端（`commands/`、`db/`、`models/`、`system/`）
- [x] 引入 SQLite：选用 **`rusqlite`**（bundled）；迁移用代码内 `PRAGMA user_version` 框架 + 启动时自动迁移（未用 `migrations/` 目录，逻辑等价、`.sql` 可移植）
- [x] 定义 **Tauri command 约定**：统一返回 `AppResult<T>`（`AppError` 序列化为 `{kind,message}`），前端 `lib/api/` 统一封装
- [x] 数据库存放路径：`app_data_dir`（`%APPDATA%\com.deskhub.app\deskhub.db`），首启自动建库建目录
- [x] 质量门禁：`cargo test`、`cargo clippy -D warnings`、`svelte-check`（prettier/eslint 暂缓，见 spec）
- [x] 全局 store 与主题骨架（亮/暗）：`stores/theme.ts` + CSS 变量，主题选择持久化到 kv

**验收：** 应用可启动；写入一条测试数据，重启后仍存在。

### M1 — Todo + 金币系统（MVP 核心）

**目标：** 完成 [项目.md](项目.md) 4.1 与 8 中的 Todo 与金币奖励闭环。

**数据模型（SQLite）**

```sql
-- 任务
CREATE TABLE todos (
  id          INTEGER PRIMARY KEY,
  title       TEXT NOT NULL,
  note        TEXT,
  done        INTEGER NOT NULL DEFAULT 0,
  due_date    TEXT,                      -- ISO8601，用于"今日任务"
  reward_coin INTEGER NOT NULL DEFAULT 10,
  created_at  TEXT NOT NULL,
  done_at     TEXT
);

-- 游戏账户（单行）
CREATE TABLE game_profile (
  id        INTEGER PRIMARY KEY CHECK (id = 1),
  coins     INTEGER NOT NULL DEFAULT 0,
  exp       INTEGER NOT NULL DEFAULT 0,
  level      INTEGER NOT NULL DEFAULT 1,
  last_tick TEXT NOT NULL                -- 用于离线收益计算
);

-- 金币流水（便于审计/防重复发奖）
CREATE TABLE coin_ledger (
  id        INTEGER PRIMARY KEY,
  amount    INTEGER NOT NULL,
  reason    TEXT NOT NULL,               -- 'todo_done' / 'idle' / ...
  ref_id    INTEGER,                     -- 关联 todo.id 等
  created_at TEXT NOT NULL
);
```

**后端 commands**
- [x] `todo_create / todo_update / todo_delete / todo_toggle_done / todo_list_today`
- [x] 完成任务时在**同一事务**内写 `coin_ledger` 并更新 `game_profile.coins`（防重复发奖：已 done 的任务再 toggle 不重复加币；取消完成不退币）
- [x] `game_get_profile`

**前端**
- [x] 今日任务列表 UI（新建/编辑/删除/勾选完成）
- [x] 金币余额展示 + 完成任务时的奖励动效（`+N🪙` 浮字）
- [x] Svelte store 与后端状态同步

> 决策：今日列表 = 全部未完成 + 今天完成；`reward_coin` 固定 10（列已预留自定义）。详见 [spec](docs/superpowers/specs/2026-06-16-deskhub-m1-todo-coins-design.md)。

**验收：** 新建→完成任务，金币增加；重启后任务与金币保留；重复勾选不重复发奖。

### M2 — 背景图片管理

**目标：** 完成 [项目.md](项目.md) 4.4 与第十章（版权来源记录）。

- [x] 选定图片来源 API：**Pexels**（免费、可商用、有官方 API），预置关键词：森林、雪山、湖泊、海边、星空
- [x] 后端用 `reqwest`(blocking) 搜索 + 下载，图片落到 `app_data_dir/backgrounds/`
- [x] **强制记录来源**（规格第十章）：

```sql
CREATE TABLE backgrounds (
  id          INTEGER PRIMARY KEY,
  local_path  TEXT NOT NULL,
  source_url  TEXT NOT NULL,            -- 原始链接，licensing 必需
  author      TEXT,
  license     TEXT,                     -- 如 'Unsplash License'
  keyword     TEXT,
  is_current  INTEGER NOT NULL DEFAULT 0,
  created_at  TEXT NOT NULL
);
```

- [x] 设置为应用背景 / 恢复默认背景（`is_current` 唯一，事务保证）
- [x] API Key 通过本地配置 `app_data_dir/config.json` 注入（应用内设置框填写），不硬编码、不入库、不进 git
- [ ] 预留扩展点：随机背景、每日自动更换、收藏、删除本地图（建表字段已支撑，未实现）

> 决策：API=Pexels；「应用背景」= 应用窗口 CSS 背景（非桌面壁纸）；当前背景以 base64 data URL 回传前端。详见 [spec](docs/superpowers/specs/2026-06-16-deskhub-m2-backgrounds-design.md)。

**验收：** 搜索→下载→设为背景成功；`backgrounds.source_url` 完整记录；重启后背景保持。

### M3 — 系统集成与打包

**目标：** 完成 [项目.md](项目.md) 4.6，使 MVP 可分发、可后台常驻。

- [x] 系统托盘：菜单（显示主窗口 / 开关 Todo·金币 widget / 退出），关闭主窗口最小化到托盘而非退出
- [x] 开机自启：`tauri-plugin-autostart`（首次默认开启、隐藏到托盘 `--hidden`、设置页可开关）
- [x] 本地通知：`tauri-plugin-notification`（测试通知按钮 + 任务到期提醒：后台 60min 轮询、kv 去重）
- [x] 数据备份：导出/导入 SQLite 文件（导出 `VACUUM INTO`；导入校验+暂存，启动时替换正库）
- [ ] 内存优化：后台空闲时降低渲染频率、按需懒加载视图（本轮降范围为核查项，未做重度改造）
- [x] 打包 Windows 安装程序（NSIS，`installMode: currentUser`），配置图标与版本号

> 第一轮范围：系统托盘 + 关闭到托盘 + 测试通知；修复 widget 窗口 capabilities 授权。详见 [spec](docs/superpowers/specs/2026-06-16-deskhub-m3-tray-notify-design.md)。
> 第二轮（收尾）：开机自启 + 数据备份 + 到期提醒 + NSIS 打包 + 设置页。详见 [spec](docs/superpowers/specs/2026-06-16-deskhub-m3-finish-design.md)、[plan](docs/superpowers/plans/2026-06-16-deskhub-m3-finish.md)。

**验收：** 安装包可在干净 Windows 上安装运行；托盘常驻；开机自启生效。

> ✅ **至此 MVP 完成**，可作为第一版发布。

### M4 — 桌面图标管理 ✅

**目标：** 完成 [项目.md](项目.md) 4.2（规格第九章列为难点）。

最终形态：**桌面半透明 widget 即图标管理面板**（策展式，非自动扫描）。

- [x] 用 `windows-rs`（IShellLink + IPersistFile）解析拖入的 `.lnk` 目标/参数
- [x] 提取真实图标（`SHGetFileInfoW → HICON → RGBA → PNG → base64`，best-effort 降级）
- [x] 拖拽添加：把桌面 `.lnk`/`.exe` 拖进 widget（`tauri://drag-drop`）→ 持久化 `custom_apps`
- [x] 编辑模式：移除 / 拖拽排序 / 重命名；普通模式点击一键启动（`ShellExecuteW`）
- [x] widget 可移动 + 右下角可缩放（尺寸/位置持久化）；主窗与托盘均可开关
- [~] 自动扫描全部桌面 + 分类/收藏：按用户决策**降范围**为策展式，未实现（字段已简化）

> 难点（快捷方式/图标读取）先 spike 编译验证再交付。详见
> [spec](docs/superpowers/specs/2026-06-16-deskhub-m4-icons-design.md)（含「widget 即管理面板」修订）、

## 三、横切关注点

- **错误处理：** 后端统一 `AppError`，前端统一 toast/日志
- **数据安全：** 敏感数据（OAuth Token、API Key）走系统凭据库，不入 SQLite 明文（规格第九章）
- **测试：** Rust 业务逻辑（发奖、离线收益结算）用 `cargo test` 覆盖；建议关键逻辑 TDD
- **性能：** 监控空闲内存占用，作为贯穿全程的非功能指标（规格第三章）
- **i18n：** 文档/规格用中文；代码标识符与注释用英文（按 CLAUDE.md 约定）
